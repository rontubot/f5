Ajustes de Conexión

Servidor de Tránsito en Railway Conectado

El backend está listo para recibir archivos QKView vía HTTPS POST.

Este token debe ser enviado en la cabecera `Authorization: Bearer ` al hacer peticiones POST al servidor.

curl -X POST -H "Authorization: Bearer BirraverdePCtoken" -F "qkview=@/ruta/al/archivo.qkview" https://myf5.up.railway.app/api/upload

Guía de Automatización en F5

Para programar la generación y subida automática del QKView desde el F5 BIG-IP, siga estos pasos:

  1. Copie el script local en el F5:

    Guarde el contenido del script `generate_qkview.sh` en el directorio `/shared/scripts/generate_qkview.sh` de su BIG-IP.

  2. Establezca permisos de ejecución: 
    chmod +x /shared/scripts/generate_qkview.sh
  3. Programe la ejecución periódica:

    Puede usar el planificador de F5 (`iCall`) o un cron job clásico. Para configurar un cron job semanal que se ejecute los domingos a la medianoche, ejecute:

    crontab -e

    Y añada la siguiente línea:

    0 0 * * 0 /shared/scripts/generate_qkview.sh > /var/log/qkview_upload.log 2>&1
El script local incluye detección del token y del hostname para etiquetar los archivos automáticamente.
